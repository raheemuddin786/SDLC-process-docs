# Structured Interviews: Predefined Questions

## Overview

**Structured Interviews** are a requirements gathering technique where interviewers ask a predetermined set of questions in a consistent order to all interviewees. This method ensures uniformity, comparability, and systematic data collection, making it ideal for gathering consistent information across multiple stakeholders.

## Definition

Structured Interviews are:
- **Systematic approach** with predefined questions
- **Consistent format** for all participants
- **Standardized process** for data collection
- **Quantitative-friendly** method for analysis
- **Repeatable technique** for multiple interviews

## Key Characteristics

### Predefined Questions
- Questions prepared in advance
- Same questions for all participants
- Consistent wording
- No deviation from script

### Standardized Format
- Fixed question order
- Uniform interview structure
- Consistent time allocation
- Same evaluation criteria

### Systematic Data Collection
- Easy to compare responses
- Quantitative analysis possible
- Pattern identification
- Statistical analysis

### Objective Approach
- Reduces interviewer bias
- Minimizes subjective interpretation
- Ensures consistency
- Enables comparison

## When to Use Structured Interviews

### Best For
- ✅ Gathering consistent data from multiple stakeholders
- ✅ Comparing responses across groups
- ✅ Quantitative analysis needs
- ✅ Large number of interviews
- ✅ Specific information requirements
- ✅ Time-constrained situations

### Not Ideal For
- ❌ Exploring unknown areas
- ❌ Building rapport
- ❌ Deep understanding of context
- ❌ Flexible conversation needed
- ❌ Complex, nuanced topics

## Structured Interview Process

### 1. Preparation Phase
- Define interview objectives
- Identify stakeholders
- Prepare question set
- Create interview guide
- Schedule interviews

### 2. Interview Phase
- Introduce purpose
- Ask predefined questions
- Record responses
- Maintain consistency
- Manage time

### 3. Analysis Phase
- Compile responses
- Identify patterns
- Analyze data
- Document findings
- Report results

## Complete Example: Customer Portal Requirements Gathering

### Project Context
**Project**: Customer Self-Service Portal Development
**Objective**: Understand customer support needs and pain points
**Interviewees**: 20 customer service representatives
**Duration**: 30 minutes per interview
**Method**: Structured interviews

### Interview Planning

#### Objectives
1. Identify current support process pain points
2. Understand customer inquiry types and frequency
3. Determine desired portal features
4. Assess technology comfort level
5. Identify success criteria

#### Stakeholder Selection
- **Primary**: Customer Service Representatives (20)
- **Secondary**: Support Managers (5)
- **Tertiary**: Customers (10)

#### Interview Schedule
- **Duration**: 30 minutes per interview
- **Location**: Conference room / Video call
- **Recording**: Audio recording + notes
- **Timeline**: 2 weeks

### Structured Interview Guide

#### Section 1: Introduction (2 minutes)

**Interviewer Script**:
"Thank you for participating in this interview. The purpose is to gather requirements for a new customer self-service portal. Your insights will help us build a system that addresses your needs and improves customer support.

This interview will take approximately 30 minutes. I'll be asking predefined questions, and I'll record your responses. All information will be kept confidential and used only for project planning.

Do you have any questions before we begin?"

**Elements**:
- **Purpose**: Set context and expectations
- **Duration**: Inform about time commitment
- **Confidentiality**: Address privacy concerns
- **Consent**: Obtain permission to proceed

---

#### Section 2: Current State Assessment (8 minutes)

**Question 1: Current Process**
"Can you describe your current process for handling customer inquiries?"

**Response Options** (Predefined categories):

**Follow-up Questions**:
- "How many inquiries do you handle per day?"

- **Question Type**: Closed-ended with options

**Expected Data**:

### Data Analysis Example

#### Quantitative Analysis

**Question 2: Pain Points (Aggregated Results from 20 Interviews)**

| Pain Point | Average Rating | % Rating 4-5 |
|------------|----------------|--------------|
| Time to resolve inquiries | 4.2 | 75% |
| Access to customer information | 3.8 | 60% |
| Knowledge base availability | 4.5 | 85% |
| Ticket tracking | 3.5 | 50% |
| Communication with customers | 3.2 | 40% |

**Analysis**:
- Knowledge base availability is the top pain point (4.5 average, 85% rate it 4-5)
- Time to resolve is second priority (4.2 average, 75% rate it 4-5)
- **Action**: Prioritize knowledge base and resolution time features

---

**Question 4: Feature Priority (Aggregated Results)**

| Feature | Average Importance | % Rating 4-5 | Top 3 Count |
|---------|-------------------|--------------|-------------|
| Order tracking | 4.6 | 90% | 18 |
| Knowledge base search | 4.5 | 85% | 15 |
| Ticket status checking | 4.3 | 80% | 12 |
| Account management | 4.1 | 75% | 8 |
| Ticket submission | 3.9 | 70% | 5 |
| FAQ section | 3.7 | 65% | 3 |
| Live chat integration | 3.5 | 55% | 2 |
| Mobile app access | 3.2 | 45% | 1 |

**Analysis**:
- Top 3 features: Order tracking, Knowledge base search, Ticket status checking
- These align with top pain points
- **Action**: Prioritize these features in development

---

#### Qualitative Analysis

**Question 2 Follow-up: Pain Point Examples**

**Theme 1: Knowledge Base Issues**
- "Customers ask the same questions repeatedly because they can't find answers"
- "We spend 30% of time answering questions that should be self-service"
- "No centralized knowledge base - information scattered across systems"

**Theme 2: Time to Resolve**
- "Need to access multiple systems to get customer information"
- "Average inquiry takes 15 minutes, but should take 5"
- "Can't quickly see customer history in one place"

**Analysis**:
- Knowledge base is critical - customers need better self-service
- Information access is fragmented - needs consolidation
- **Action**: Design unified knowledge base and customer information dashboard

---

### Findings Summary

#### Key Findings
1. **Top Pain Points**: Knowledge base availability (4.5/5), Time to resolve (4.2/5)
2. **Priority Features**: Order tracking, Knowledge base search, Ticket status checking
3. **Critical Information**: Order history, Previous tickets, Account activity
4. **Key Integration**: Ticket system, Customer database
5. **Success Metric**: 30% reduction in inquiry volume

#### Recommendations
1. **Phase 1 Features**: Order tracking, Knowledge base, Ticket status
2. **Integration Priority**: Ticket system, Customer database
3. **Training**: Basic portal navigation (2-hour session)
4. **Success Criteria**: 30% inquiry reduction within 6 months
